# CI/CD Secrets Setup Guide

Complete guide for setting up CI/CD secrets for the Flutter multi-platform app.

## Overview

**Total: 7 Secrets**

| Category | Secret Name | Generated By |
|----------|-------------|--------------|
| **Fastlane (1)** | `FASTLANE_ENV_FILE` | `encode_env_for_cicd.sh` |
| **iOS (4)** | `IOS_DEV_PROFILE`<br>`IOS_STAGING_PROFILE`<br>`IOS_SANDBOX_PROFILE`<br>`IOS_PROD_PROFILE` | `encode_for_cicd.sh` |
| **Android (1)** | `ANDROID_KEYSTORE_PROPERTIES` | Manual: `cat keystore.properties \| base64` |
| **App Config (1)** | `APP_CONFIG` | Manual: `cat apps/main/.env \| base64` |

**Security Strategy:**
- ✅ Android `.jks` keystores → **Committed** (encrypted by passwords)
- 🔒 Android `keystore.properties` → **Secret** (contains passwords)
- 🔒 iOS provisioning profiles → **Secret** (change frequently)
- 🔒 Fastlane `.env` → **Secret** (API keys & tokens)
- 🔒 App `.env` → **Secret** (API endpoints, feature flags, environment config)

## Quick Start

### Step 1: Encode Secrets

```bash
# 1. Fastlane environment variables
cd apps/main/fastlane
sh encode_env_for_cicd.sh

# 2. iOS provisioning profiles
cd ../ios
sh encode_for_cicd.sh

cd ../android/keystores
cat keystore.properties | base64

# 4. App configuration
cd ../..
cat .env | base64
```

### Step 2: Add to CI/CD

Copy the 6 outputs to your CI/CD platform (GitHub Actions or GitLab CI).

## Fastlane Secrets

**Secret:** `FASTLANE_ENV_FILE`  
**File:** `apps/main/fastlane/.env`  
**Script:** `encode_env_for_cicd.sh`

**Setup:**
```bash
cd apps/main/fastlane
cp .env.example .env
# Edit .env with your values
```

**Required in `.env`:**
- 8 Firebase App IDs (4 Android + 4 iOS)
- Firebase CLI token: `firebase login:ci`

## Android Signing Secrets

**Secret:** `ANDROID_KEYSTORE_PROPERTIES`  
**File:** `apps/main/android/keystores/keystore.properties`

**Setup:**
```bash
cd apps/main/android/keystores
cat keystore.properties | base64  # Copy output to CI/CD secret
```

**Security Strategy:**
- ✅ `.jks` keystores → **Committed to git** (encrypted by passwords, safe)
- 🔒 `keystore.properties` → **Secret** (contains passwords)

**Why this works:** Keystores are encrypted containers. Without passwords, they're useless. Committing keystores simplifies CI/CD (1 secret vs 5) and makes rotation easier.

**Alternative:** Keep keystores as secrets too (5 total Android secrets) for extra paranoia. Only needed for public repos.

## GitHub Actions Setup

Add 7 secrets in **Settings → Secrets and variables → Actions → New repository secret**

```yaml
name: Deploy

on:
  push:
    branches: [develop]

jobs:
  deploy-android:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup Flutter
        uses: subosito/flutter-action@v2
        
      - name: Setup Ruby
        uses: ruby/setup-ruby@v1
        with:
          bundler-cache: true
          working-directory: apps/main/fastlane
      
      - name: Setup Fastlane .env file
        working-directory: apps/main/fastlane
        run: echo "${{ secrets.FASTLANE_ENV_FILE }}" | base64 -d > .env
      
      - name: Setup Android signing
        working-directory: apps/main/android/keystores
        run: echo "${{ secrets.ANDROID_KEYSTORE_PROPERTIES }}" | base64 -d > keystore.properties
      
      - name: Build and Deploy
        run: |
          sh ./distribution.sh \
            -a main \
            -e dev \
            -p android \
            --app-config "${{ secrets.APP_CONFIG }}"

  deploy-ios:
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup Flutter
        uses: subosito/flutter-action@v2
        
      - name: Setup Ruby
        uses: ruby/setup-ruby@v1
        with:
          bundler-cache: true
          working-directory: apps/main/fastlane
      
      - name: Setup Fastlane .env file
        working-directory: apps/main/fastlane
        run: echo "${{ secrets.FASTLANE_ENV_FILE }}" | base64 -d > .env
      
      - name: Setup iOS signing
        run: |
          echo "${{ secrets.IOS_DEV_PROFILE }}" | base64 -d > dev.mobileprovision
          echo "${{ secrets.IOS_STAGING_PROFILE }}" | base64 -d > staging.mobileprovision
          echo "${{ secrets.IOS_SANDBOX_PROFILE }}" | base64 -d > sandbox.mobileprovision
          echo "${{ secrets.IOS_PROD_PROFILE }}" | base64 -d > prod.mobileprovision
          open dev.mobileprovision
          open staging.mobileprovision
          open sandbox.mobileprovision
          open prod.mobileprovision
      
      - name: Build and Deploy
        run: |
          sh ./distribution.sh \
            -a main \
            -e dev \
            -p ios \
            --app-config "${{ secrets.APP_CONFIG }}"
```

## GitLab CI Setup

Add 7 variables in **Settings → CI/CD → Variables → Add variable**

**Important:** Set as:
- **Type:** Variable (not File)
- **Protected:** Yes (for production)
- **Masked:** Yes (to hide in logs)

```yaml
stages:
  - build_deploy

.setup_fastlane:
  before_script:
    - gem install bundler
    - cd apps/main/fastlane && echo "$FASTLANE_ENV_FILE" | base64 -d > .env && cd ../../..

.setup_android_signing:
  before_script:
    - cd apps/main/android/keystores
    - echo "$ANDROID_KEYSTORE_PROPERTIES" | base64 -d > keystore.properties
    - cd ../../..

.setup_ios_signing:
  before_script:
    - echo "$IOS_DEV_PROFILE" | base64 -d > dev.mobileprovision
    - echo "$IOS_STAGING_PROFILE" | base64 -d > staging.mobileprovision
    - echo "$IOS_SANDBOX_PROFILE" | base64 -d > sandbox.mobileprovision
    - echo "$IOS_PROD_PROFILE" | base64 -d > prod.mobileprovision
    - open dev.mobileprovision
    - open staging.mobileprovision
    - open sandbox.mobileprovision
    - open prod.mobileprovision

deploy_android_dev:
  stage: build_deploy
  image: mingc/android-build-box:latest
  extends:
    - .setup_fastlane
    - .setup_android_signing
  script:
    - sh ./distribution.sh -a main -e dev -p android
  only:
    - develop

deploy_ios_dev:
  stage: build_deploy
  tags:
    - macos
  extends:
    - .setup_fastlane
    - .setup_ios_signing
  script:
    - sh ./distribution.sh -a main -e dev -p ios
  only:
    - develop
```

## Security Best Practices

**Never commit:**
```gitignore
apps/main/fastlane/.env
apps/main/android/keystores/keystore.properties
apps/main/ios/signing_res/**/*.mobileprovision
*.p12
*.cer
```

**Rotate regularly:**
- Firebase tokens (expire periodically)
- Provisioning profiles (yearly)
- App Store Connect API keys (annually)

## Troubleshooting

**Verify secret decoding:**
```bash
# Fastlane
echo "$FASTLANE_ENV_FILE" | base64 -d > .env && cat .env

# Android
echo "$ANDROID_KEYSTORE_PROPERTIES" | base64 -d > keystore.properties
ls -la apps/main/android/keystores/*.jks  # Verify keystores exist

# iOS
echo "$IOS_DEV_PROFILE" | base64 -d > test.mobileprovision
security cms -D -i test.mobileprovision  # View profile details
```

**Base64 encoding differences:**
- macOS/Linux: `cat file | base64` and `echo "$SECRET" | base64 -d > file`
- Some CI: May need `-w 0` flag or `--decode` instead of `-d`
