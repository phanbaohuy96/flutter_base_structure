## Unreleased

* **Breaking — `ListingUseCase`:** the use case now tracks only its pagination
  cursor and no longer retains the loaded items. Removed the public `pagination`
  and `data` getters; use `canNext` instead of `pagination.canNext`, and read the
  accumulated items from the consuming bloc's state. The `listing` module
  generator template already emits the new API — **regenerate listing modules
  (`make gen_main`) or replace `_listingUsecase.pagination.canNext` with
  `_listingUsecase.canNext`** in code generated by an older template.
* **Breaking — removed `Pagination` entity:** `domain/entity/pagination.dart`
  was unused after the `ListingUseCase` change and has been deleted. Track
  listing state through `ListingUseCase`/the bloc instead.
